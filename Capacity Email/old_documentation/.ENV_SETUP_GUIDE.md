# üìß Email Setup Using .env File

## Quick Setup (2 Steps)

### Step 1: Edit `.env` File
1. Open `.env` file in the project directory
2. Replace the values:
   ```
   GMAIL_SENDER_EMAIL=your-email@gmail.com
   GMAIL_SENDER_PASSWORD=your-app-password
   SCRUM_MASTER_EMAIL=scrum-master@company.com
   ```

### Step 2: Get Gmail App Password
1. Visit https://myaccount.google.com/security
2. Enable "2-Step Verification" (if not done)
3. Go to "App passwords"
4. Select "Mail" and "Windows Computer"
5. Copy the 16-character password
6. Paste it in `.env` file as `GMAIL_SENDER_PASSWORD`

### Step 3: Test
```bash
py sprint_capacity_app.py --analyze
```

That's it! ‚úÖ

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Email configuration incomplete" | Check `.env` file values are not empty |
| "Authentication failed" | Verify App Password (not Gmail password) |
| ".env file not found" | Make sure `.env` is in the same directory as `sprint_capacity_app.py` |
| Changes not taking effect | Restart Python/IDE to reload `.env` |

## Common Questions

**Q: Can I use my Gmail password?**  
A: No, use App Password instead. Gmail requires App Passwords for security.

**Q: Can I share the `.env` file?**  
A: No, it has sensitive credentials. Each person keeps their own `.env`.

**Q: Is `.env` file safe?**  
A: Yes, it's in `.gitignore` so it won't be committed to git.

**Q: Can I use multiple email addresses?**  
A: Currently, only one sender email. For multiple, edit `GMAIL_SENDER_EMAIL`.
